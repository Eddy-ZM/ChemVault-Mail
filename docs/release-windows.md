# Release Windows Desktop

This project publishes ChemVault Mail for Windows outside Microsoft Store. Distribution and app updates use GitHub Releases.

## Version Rule

The desktop app version is `mail-vue/package.json`.

Current version:

```text
0.1.0
```

Installer artifact:

```text
ChemVault-Mail-Setup-0.1.0.exe
```

For every release:

1. Update `mail-vue/package.json` version.
2. Keep the repo root `package.json` version aligned.
3. Update docs or release notes if the user-visible update changed.
4. Commit the change.
5. Tag the commit as `vX.Y.Z`.

The GitHub Actions workflow rejects tag builds where `vX.Y.Z` does not match `mail-vue/package.json`.

## Build Locally

```powershell
pnpm --dir mail-vue install --frozen-lockfile
npm run desktop:dist:win
```

Check these files:

```text
mail-vue/release/windows/ChemVault-Mail-Setup-0.1.0.exe
mail-vue/release/windows/ChemVault-Mail-Setup-0.1.0.exe.blockmap
mail-vue/release/windows/latest.yml
```

Then run the Windows smoke test:

```powershell
npm run desktop:test:win
```

## Publish With GitHub Actions

Create and push a tag:

```powershell
git tag v0.1.0
git push origin v0.1.0
```

`.github/workflows/build-windows.yml` runs on `v*` tags. It:

1. Uses `windows-latest`.
2. Installs pnpm and Node.js.
3. Installs `mail-vue` dependencies.
4. Builds the Vite desktop renderer.
5. Builds the NSIS installer.
6. Uploads the installer files as workflow artifacts.
7. Creates or updates the GitHub Release.
8. Uploads `.exe`, `.blockmap`, and `latest.yml`.

The default `GITHUB_TOKEN` is used by the workflow. Do not hardcode a personal access token.

## Publish Locally

Local publish requires a GitHub token in the shell environment:

```powershell
$env:GH_TOKEN='<github token with release permission>'
pnpm --dir mail-vue desktop:publish:win
```

Do not commit the token or write it into `.env` files.

## Update Source

electron-updater reads the GitHub Release metadata generated by electron-builder. The key file is:

```text
latest.yml
```

It includes the latest version, installer file name, hash, and blockmap metadata. The packaged app compares this manifest against the installed app version.

## Test Auto Update

Recommended update test:

1. Build and install version `0.1.0`.
2. Launch it once and confirm no update error blocks startup.
3. Bump `mail-vue/package.json` and root `package.json` to `0.1.1`.
4. Build and publish tag `v0.1.1`.
5. Launch the installed `0.1.0` app.
6. Confirm it detects `0.1.1`.
7. Choose download.
8. Confirm the app stays usable during download.
9. Choose "Restart and update".
10. Confirm the app restarts as `0.1.1`.

For a local manifest recognition test before publishing, run the packaged app with `CHEMVAULT_DESKTOP_UPDATE_FEED_URL` pointed at a local electron-updater generic feed that serves `latest.yml`. The app accepts local `http://localhost` feeds only for QA; production feed overrides must be HTTPS.

Update failures must leave the installed version usable. Inspect `desktop-updater.log` in the Electron app logs directory if checks fail.

## Rollback

For new downloads, mark the previous stable GitHub Release as latest or update the download page URL.

For already installed apps, electron-updater does not downgrade by default. Publish a higher patch version that reverts the bad change, for example `0.1.2`, and mark that release as latest.

## Code Signing

Unsigned installers are supported but can trigger Windows SmartScreen.

To sign in GitHub Actions, add repository secrets:

```text
WINDOWS_CODESIGN_CERTIFICATE
WINDOWS_CODESIGN_PASSWORD
```

The certificate should be an EV/OV code signing certificate usable by electron-builder. Never commit certificate files or passwords.

## Manual Release Checklist

- `mail-vue/package.json` version matches the tag.
- `package.json` version matches the desktop version.
- No backend secrets are present in desktop files or release artifacts.
- `latest.yml` is uploaded with the installer.
- Download page points at the intended GitHub Release channel.
- Installer warning is documented if unsigned.
- Login, inbox, read mail, compose, external links, update check, and uninstall are tested on Windows 10 or Windows 11.
- `npm run desktop:test:win` passes on a clean installer build.
